# JSON模块提取工具测试报告

## 测试概述
本报告总结了对JSON模块提取工具的全面测试结果，包括基础功能、高级功能和性能测试。

## 测试用例分类

### 基础功能测试

#### 1. simple_3nodes.json (简单线性结构)
- **测试内容**: 3个节点的线性流程 (inject → function → debug)
- **测试参数**: 节点数范围 2-3
- **结果**: ✅ 成功
  - 生成了2个节点的子图 (func1 → debug1)
  - 生成了3个节点的完整子图 (inject1 → func1 → debug1)
- **验证**: 连接关系正确保留

#### 2. cycle_4nodes.json (循环结构)
- **测试内容**: 4个节点的环形连接 (A→B→C→D→A)
- **测试参数**: 节点数范围 3-4
- **结果**: ✅ 成功
  - 生成了4个节点的完整循环子图
- **验证**: 循环结构正确保留

### 高级功能测试

#### 3. mixed_connections.json (混合连接)
- **测试内容**: 包含wires、z属性、server属性的复杂连接
- **测试参数**: 节点数范围 4-6，6个总节点
- **结果**: ✅ 成功
  - 生成了5个节点和6个节点的子图
- **验证**: 所有连接类型（有向wires、无向z属性、server引用）都正确保留

#### 4. ignore_z_property.json (忽略属性功能)
- **测试内容**: 7个节点，都通过z属性连接到tab1
- **测试参数**: 节点数范围 3-4，忽略z属性
- **结果**: ✅ 成功
  - 生成了2个3节点的独立子图
- **验证**: 忽略z属性后，节点按wires重新分组

#### 5. nested_groups.json (嵌套组结构)
- **测试内容**: 包含group和z属性的嵌套结构
- **测试参数**: 节点数范围 3-4，7个总节点
- **结果**: ✅ 成功
  - 生成了4个节点的子图
- **验证**: 正确处理了组和嵌套关系

### 性能测试

#### 6. large_complex_graph.json (大型复杂图)
- **测试内容**: 21个节点的复杂流程图，包含多种节点类型和连接
- **测试参数**: 节点数范围 5-8
- **结果**: ✅ 成功
  - 生成了大量子图：
    - 5节点子图: 54个
    - 6节点子图: 89个
    - 7节点子图: 88个
    - 8节点子图: 89个
  - 总计: 320个子图
- **验证**: 大规模处理能力良好，所有子图都保持了正确的连接关系

#### 7. edge_cases.json (边界情况测试)
- **测试内容**: 包含各种边界情况的6个节点
- **测试参数**: 节点数范围 1-3
- **结果**: ✅ 成功
  - 生成了5个1节点子图和1个2节点子图
  - 未找到3节点子图（符合预期）
- **详细结果**:
  - `module_edge_cases_1_1.json`: 单个孤立节点 (single, type: inject, 无连接)
  - `module_edge_cases_1_2.json`: 单个孤立节点 (isolated1, type: function, 无连接)
  - `module_edge_cases_1_3.json`: 单个孤立节点 (isolated2, type: debug, 无连接)
  - `module_edge_cases_1_4.json`: 单个节点 (pair2, type: debug, 无连接)
  - `module_edge_cases_1_5.json`: 自环节点 (self_loop, type: function, 连接到自身)
  - `module_edge_cases_2_1.json`: 2节点连接 (pair1 → pair2)
- **验证**: 正确处理了孤立节点、自环、简单连接等边界情况
